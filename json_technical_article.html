一、什麼是 JSON？

JSON，全名是 JavaScript Object Notation，中文常稱為「JavaScript 物件表示法」。它是一種輕量級的資料交換格式，主要用來在不同系統、程式、網站、API、資料庫與 AI Agent 之間傳遞資料。

雖然 JSON 的名字裡有 JavaScript，但它並不只屬於 JavaScript。現在幾乎所有主流程式語言都支援 JSON，例如 Python、Java、PHP、C#、Go、Ruby、Swift、Kotlin 等。

簡單來說，JSON 就像是一種「程式之間都看得懂的資料表達方式」。

二、為什麼 JSON 很重要？

在現代數位系統中，資料不再只存在單一程式裡，而是會在許多地方之間流動。

前後端資料交換前端網頁向後端伺服器請求資料，常使用 JSON 作為回傳格式。
App 與雲端服務手機 App 讀取雲端資料時，JSON 是常見格式。
AI Agent 工具呼叫Agent 需要明確的參數與輸出格式，JSON 很適合承載。
自動化工作流任務設定、流程狀態、資料欄位，都可以用 JSON 表示。

JSON 的優點包括結構清楚、容易閱讀、檔案體積小、容易被程式解析、適合 API 傳輸，也適合設定檔與資料交換。

三、JSON 的基本結構

JSON 主要由兩種核心結構組成：物件 Object 與陣列 Array。

物件 Object：使用大括號 { }
{
  "name": "黑熊老師",
  "subject": "自然科學",
  "level": "國小"
}
陣列 Array：使用中括號 [ ]
[
  "自然科學",
  "AI 教學",
  "科展指導"
]

物件適合表示一筆有欄位名稱的資料，陣列則適合表示多筆資料或清單。

四、JSON 的資料型態

JSON 支援的常見資料型態
{
  "string": "這是文字",
  "number": 168,
  "boolean": true,
  "nullValue": null,
  "array": ["觀察", "實驗", "紀錄"],
  "object": {
    "teacher": "黑熊老師",
    "subject": "自然"
  }
}
型態說明範例
string文字"自然科學"
number數字100、3.14
boolean布林值true、false
null空值null
array陣列["A", "B", "C"]
object物件{"name": "小明"}

五、JSON 的語法規則

  1. 鍵名必須使用雙引號。
  2. 字串必須使用雙引號。
  3. 不能使用單引號。
  4. 最後一個項目後面不能加逗號。
  5. 註解不屬於標準 JSON。
  6. true、false、null 必須使用小寫。
正確寫法
{
  "name": "黑熊老師",
  "active": true,
  "score": 100
}
錯誤寫法
{
  name: '黑熊老師',
  active: True,
  score: 100,
}

六、JSON 在 API 中的應用

JSON 最常見的應用場景之一就是 API。例如一個查詢學生資料的 API，可能會回傳以下內容：

學生資料 API 回傳範例
{
  "studentId": "S001",
  "name": "陳小明",
  "grade": 5,
  "class": "五年甲班",
  "scores": {
    "science": 95,
    "math": 88,
    "chinese": 92
  }
}

前端網站或 App 收到這份 JSON 後，就可以把資料顯示成學生姓名、年級、班級與各科成績。

七、JSON 在設定檔中的應用

許多工具會使用 JSON 作為設定檔格式，例如網站專案、AI Agent、VS Code、npm、Firebase、Supabase 等。

網站設定檔範例
{
  "siteName": "黑熊老師自然科學教材中心",
  "theme": "light",
  "language": "zh-TW",
  "features": {
    "search": true,
    "darkMode": true,
    "copyButton": true
  }
}

這種方式的好處是：人可以看懂，程式也可以讀懂。

八、JSON 在 AI Agent 中的應用

在 AI Agent 工作流中，JSON 很重要。因為 Agent 通常需要明確知道要執行什麼任務、使用哪些工具、輸入資料是什麼、輸出格式是什麼，以及成功或失敗條件。

AI Agent 任務描述範例
{
  "task": "generate_lesson_plan",
  "subject": "自然科學",
  "grade": "五年級",
  "topic": "熱的傳播",
  "outputFormat": ["docx", "html"],
  "requirements": {
    "includeObjectives": true,
    "includeActivities": true,
    "includeAssessment": true,
    "language": "zh-TW"
  }
}

這種寫法比自然語言更明確，也更容易讓 Agent 自動化執行。

十、常見錯誤與排除方式

1. 多了一個逗號

錯誤
{
  "name": "黑熊老師",
  "subject": "自然",
}
修正
{
  "name": "黑熊老師",
  "subject": "自然"
}

2. 使用單引號

錯誤
{
  'name': '黑熊老師'
}
修正
{
  "name": "黑熊老師"
}

3. 註解放進標準 JSON

錯誤
{
  // 這是網站名稱
  "siteName": "Science Portal"
}

標準 JSON 不支援註解。如果需要註解，可以改用 JSONC，或另外撰寫說明文件。

十一、JSON Schema 是什麼？

JSON Schema 可以用來規範 JSON 的資料格式，就像是 JSON 的「格式檢查表」。

JSON Schema 範例
{
  "type": "object",
  "required": ["course", "grade", "unit"],
  "properties": {
    "course": {
      "type": "string"
    },
    "grade": {
      "type": "number"
    },
    "unit": {
      "type": "string"
    }
  }
}

這樣可以避免資料格式混亂，例如少了欄位、型態錯誤或命名不一致。

十二、結語

JSON 看似只是簡單的資料格式，但它其實是現代數位系統的重要基礎。從網站前後端、API、資料庫、設定檔，到 AI Agent 工作流，JSON 都扮演著關鍵角色。

對教師、開發者與 AI 工作者來說，學會 JSON 不只是學會一種語法，而是學會如何把資料整理成「人看得懂、機器也能處理」的結構。

在 AI 時代，能把想法轉成清楚的 JSON 結構，就等於多了一種和系統、工具、Agent 溝通的能力。
