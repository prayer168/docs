1. React 的定義：它不是完整框架，而是 UI 函式庫

React 是一套用來建立 Web 與原生使用者介面的 JavaScript 函式庫。它讓開發者把畫面拆成可重複使用的「元件」，並以資料與狀態描述介面應該呈現的樣子。

最精準的理解：React 主要解決的是「介面如何由資料產生、如何在資料改變時可靠更新」；路由、後端、資料庫、帳號、部署與完整專案架構通常由其他工具補上。

React 負責

  • 建立 UI 元件
  • 處理狀態與事件
  • 根據資料重新渲染
  • 組合畫面與互動流程

React 本身不完整負責

  • 網址路由與 SEO
  • 後端 API 與資料庫
  • 帳號與權限
  • 建置、部署與伺服器架構

一句話公式

UI = f(props, state, context) 畫面 = 元件根據「外部資料、內部狀態、共享環境」計算出的結果

2. React 在產品開發中扮演什麼角色？

React 位於「使用者看得到、點得到、操作得到」的前端介面層。它把設計稿、資料與使用者操作連接起來，是網站或 App 的互動中樞。

使用者 ↓ 點擊、輸入、拖曳、選擇 React UI 元件層 ↓ 呼叫 API／AI 模型／Supabase／Firebase／自建後端 ↓ 回傳 JSON、圖片、影音、學習紀錄 React 更新狀態並重新呈現畫面
技術主要角色與 React 的關係
HTML內容結構與語意React 最終仍產生或控制 DOM 結構
CSS／Tailwind視覺樣式與 RWD負責元件外觀
JavaScript／TypeScript程式邏輯與資料型別React 元件的程式語言
Next.js全端 React 框架補上路由、伺服器渲染、資料載入、SEO 與部署架構
React Native原生行動 App UI沿用 React 思維，但輸出為 Android／iOS 原生元件
Supabase資料庫、登入、儲存、即時資料React 作為前端介面呼叫服務

3. React 的核心原理

元件化

把大型畫面拆成按鈕、卡片、題目、圖表、導覽列等小元件，再組成頁面。

宣告式 UI

描述「某個狀態下畫面應該長什麼樣」，而不是逐步下指令修改每個 DOM。

單向資料流

資料通常由父元件往下傳，事件由子元件回報，讓狀態來源更容易追蹤。

3.1 元件 Component

現代 React 主要使用函式元件。每個元件接收 props，並回傳 JSX。

ScienceCard.jsx
function ScienceCard({ title, description }) {
  return (
    <article className="science-card">
      <h2>{title}</h2>
      <p>{description}</p>
    </article>
  );
}

3.2 JSX

JSX 是 JavaScript 的語法擴充，讓開發者以接近 HTML 的方式描述 UI。它不是字串，也不是瀏覽器直接理解的 HTML；建置工具會將它轉換成 JavaScript。

3.3 Props 與 State

概念用途例子
props父元件傳入的唯讀資料題目文字、學生姓名、卡片標題
state元件需要記住且會變動的資料目前答案、分數、頁碼、是否顯示提示
context跨多層元件共享資料語言、登入者、主題色、課程資訊

3.4 純渲染與不可變更新

React 元件應盡量保持純粹：相同 props、state、context 應產生相同輸出。更新物件或陣列狀態時，通常建立新值，而不是直接修改原物件，讓更新更可預測。

4. React 如何渲染與更新畫面？

1. 觸發
初次載入或 setState
→
2. Render
呼叫元件計算 UI
→
3. 比較
計算需要改變之處
→
4. Commit
更新必要的 DOM

「Virtual DOM」是常見簡化說法，但更完整的理解是：React 會重新呼叫相關元件，取得新的 UI 描述，經過協調與差異計算後，在提交階段對真實 DOM 執行必要更新。

重要：重新渲染不等於整個網頁全部重畫。React 可能重新執行元件函式，但提交到 DOM 的通常只是必要變更。

Key 為什麼重要？

渲染清單時，key 協助 React 辨識每一筆資料的身分，決定狀態該保留、移動或重設。應使用穩定且唯一的資料 ID，避免任意使用陣列索引。

5. Hooks：讓函式元件使用 React 能力

Hook用途教材例子
useState保存元件狀態選擇答案、計分、開關提示
useEffect同步外部系統讀取 API、計時器、事件訂閱
useRef保留值或取得 DOM聚焦輸入框、控制影片
useContext共享跨層資料全站語言、登入、課程主題
useReducer管理複雜狀態流程多步驟實驗、題組作答流程
useMemo快取昂貴計算大量實驗資料統計
簡易計數器
import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <section>
      <p>觀察次數：{count}</p>
      <button onClick={() => setCount(count + 1)}>
        增加一次
      </button>
    </section>
  );
}
useEffect 的正確觀念：它主要用來讓元件與 React 外部系統同步，例如網路連線、第三方元件、瀏覽器 API 或訂閱；不應把所有一般資料計算都塞進 Effect。

6. React 的典型應用場景

互動網站

搜尋、篩選、登入、購物車、動態表單、會員中心。

管理系統

儀表板、資料表格、權限管理、即時監控、報表。

AI 應用

聊天介面、生成器、Agent 工作台、RAG 知識庫、串流回覆。

數位教材

互動模擬、測驗、學習路徑、圖表、動畫、學習紀錄。

PWA

可安裝、離線、跨平台的 Web App。

原生 App

搭配 React Native 開發 Android 與 iOS 應用。

React 特別適合的情況

  • 畫面包含大量會改變的資料或互動狀態。
  • 同一個 UI 元件會在多個頁面重複使用。
  • 需要多人協作與清楚的元件邊界。
  • 未來可能擴充登入、資料庫、AI、圖表或跨平台 App。

7. React 在數位教材與自然科教學中的角色

React 可以把「靜態教材頁」升級為「可操作、可回饋、可記錄、可個人化」的學習系統。教師提供科學內容與教學流程，React 負責把學習活動轉化為互動介面。

教材功能React 的作用自然科例子
互動模擬根據滑桿、按鈕即時改變模型月相、電路、熱傳導、槓桿
探究紀錄保存觀察值、步驟與照片狀態昆蟲調查、植物生長、雨量紀錄
形成性評量題目元件、即時回饋、分數計算酸鹼判斷、電流方向、分類活動
資料視覺化將資料轉成圖表或比較表溫度曲線、聲音振幅、物種數量
差異化教學依程度顯示不同提示與任務基礎、進階、挑戰三層題目
AI 助教串接模型 API 並呈現串流答案引導提問、實驗建議、資料解釋

數位教材的元件拆分示例

ScienceLessonPage ├── LessonHeader ├── ConceptCard ├── InteractiveSimulation │ ├── ControlSlider │ ├── ModelCanvas │ └── ObservationPanel ├── InquiryWorksheet ├── QuizSection │ ├── QuestionCard │ └── FeedbackBox └── LearningProgress
教育價值：元件化不只方便寫程式，也對應教學設計。概念說明、操作、觀察、推理、回饋與評量可以各自封裝，再依不同單元重新組合。

8. React 在網站與 Web App 開發中的應用

8.1 純 React 單頁應用 SPA

SPA 通常先載入一份主要網頁文件，再由 JavaScript 更新內容。適合後台、工具、儀表板與登入後系統。

8.2 搭配 Next.js 的內容型或全端網站

Next.js 是 React 框架，提供檔案路由、伺服器與客戶端元件、資料載入、SEO、影像最佳化及部署等能力。教材入口網站、公開課程網站與需要搜尋引擎收錄的內容，通常比純 SPA 更適合採用 Next.js。

8.3 PWA

PWA 使用 Web 技術，但可提供接近平台 App 的體驗，例如安裝到主畫面、離線使用及背景能力。對校園平板、戶外調查與網路不穩的教材特別有價值。

適合純 React＋Vite

  • 內部工具
  • 登入後儀表板
  • 小型互動模擬
  • 靜態主機部署

適合 Next.js

  • 公開教材入口
  • SEO 與社群分享
  • 大量內容頁
  • 需要伺服器功能或全端整合

9. React 在 Android／iOS App 開發中的應用

React Native 將 React 的元件、props、state 與 Hooks 思維帶入原生行動開發。它不是把網頁塞進 App，而是使用 React 描述介面，再映射到 Android 與 iOS 的原生視圖元件。

React Native 範例
import { View, Text, Pressable } from 'react-native';
import { useState } from 'react';

export default function ObservationApp() {
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>已觀察昆蟲：{count} 隻</Text>
      <Pressable onPress={() => setCount(count + 1)}>
        <Text>新增紀錄</Text>
      </Pressable>
    </View>
  );
}

App 教材可加入

相機拍照GPS 定位離線紀錄推播提醒麥克風錄音感測器資料

例如「校園生物多樣性調查 App」可用相機拍攝、GPS 標記位置、離線保存，回到教室後同步至雲端資料庫。

10. 完整示例：酸鹼指示劑互動教材

下例示範 React 如何管理選項、計算結果、提供即時回饋，並將資料驅動的教材內容拆成可維護元件。

AcidBaseLab.jsx
import { useState } from 'react';

const samples = [
  { id: 1, name: '檸檬汁', type: '酸性', color: '紅色' },
  { id: 2, name: '食鹽水', type: '中性', color: '紫色' },
  { id: 3, name: '小蘇打水', type: '鹼性', color: '藍綠色' },
];

function SampleButton({ sample, onChoose }) {
  return (
    <button onClick={() => onChoose(sample)}>
      {sample.name}
    </button>
  );
}

export default function AcidBaseLab() {
  const [selected, setSelected] = useState(null);
  const [guess, setGuess] = useState('');

  const isCorrect = selected && guess === selected.type;

  return (
    <main>
      <h1>紫色高麗菜酸鹼實驗</h1>

      <section>
        {samples.map(sample => (
          <SampleButton
            key={sample.id}
            sample={sample}
            onChoose={setSelected}
          />
        ))}
      </section>

      {selected && (
        <section>
          <h2>預測：{selected.name} 是哪一類？</h2>
          {['酸性', '中性', '鹼性'].map(option => (
            <button key={option} onClick={() => setGuess(option)}>
              {option}
            </button>
          ))}

          {guess && (
            <p role="status">
              {isCorrect
                ? `答對了！指示劑約呈 ${selected.color}。`
                : '再想想：觀察顏色變化與酸鹼性之間的關係。'}
            </p>
          )}
        </section>
      )}
    </main>
  );
}

此示例包含的 React 觀念

  • 資料陣列驅動畫面，而不是重複手寫三組 HTML。
  • 使用 key 辨識每個樣本與選項。
  • 父元件保存目前樣本與答案狀態。
  • 條件渲染控制題目與回饋何時出現。
  • 以 props 與 callback 連接子元件。
  • role="status" 改善輔助科技對即時回饋的感知。

12. 測試、無障礙、效能與安全

12.1 測試

React Testing Library 強調以接近使用者操作的方式查找與互動元件，而不是緊盯元件內部實作。教材應測試「學生是否看得到題目、能否按鈕作答、回饋是否正確」。

12.2 無障礙

React 不會自動保證無障礙。仍需使用正確 HTML 語意、鍵盤可操作、替代文字、表單標籤、足夠對比與清楚焦點，並依 WCAG 原則設計。數位教材尤其要考量視覺、聽覺、肢體、認知與閱讀差異。

12.3 效能

  • 先測量再最佳化，不要過早大量使用 memo。
  • 大型清單可採分頁或虛擬化。
  • 非必要功能可使用 lazy 與 Suspense 延遲載入。
  • 避免重複、矛盾或可由其他狀態計算出的 state。
  • 圖片、影片與 3D 模型應壓縮並按需載入。

12.4 安全

  • 不要把私密 API Key 寫在前端程式。
  • AI 回傳內容與學生輸入需驗證、過濾與權限控管。
  • 後端資料庫必須設定存取規則；不能只靠隱藏按鈕。
  • 避免不必要使用 dangerouslySetInnerHTML。

13. 何時不一定需要 React？

React 很強，但不是所有網站都必須使用。

情況更簡單的選擇
只有少量文字、圖片、沒有互動純 HTML＋CSS
單一活動頁，只需幾個簡單按鈕原生 JavaScript
內容管理為主，現有 CMS 已足夠WordPress／其他 CMS
極小型嵌入式互動Web Components 或輕量工具
常見誤區：為了「技術看起來新」而使用 React，可能增加建置、依賴、維護與除錯成本。選型應從互動複雜度、團隊能力、內容更新方式與長期維護出發。

14. 建議學習與實作路線

  1. HTML、CSS、JavaScript 基礎：DOM、事件、陣列方法、模組、非同步。
  2. React 基礎：元件、JSX、props、state、事件、條件與清單渲染。
  3. Hooks 與設計：useState、useEffect、狀態提升、元件拆分。
  4. TypeScript：為 props、API 資料與教材模型加上型別。
  5. 路由與資料：React Router 或 Next.js、Fetch、Supabase。
  6. 品質：測試、無障礙、RWD、效能、安全。
  7. 專案：完成一套可部署的互動自然科教材。

推薦第一個專案

「校園生物觀察教材」：包含物種卡片、搜尋篩選、拍照紀錄、地點、觀察次數、圖表、學習單與教師後台。先做 Web 版，再視需求擴展為 PWA 或 React Native App。
