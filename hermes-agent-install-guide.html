Hermes Agent 安裝技術文件

本文件整理這次在 Windows 上安裝 Hermes Agent 的完整過程，包含 CLI 安裝、Desktop 啟動、Gateway / Provider 設定、Git stash 衝突、GPU 啟動錯誤，以及最後如何建立含 Hermes Logo 的桌面捷徑。

適用系統Windows 10 / 11
安裝型態CLI + Desktop
主要問題Git 衝突、GPU、Provider
文件版本2026-07-03

一、環境與安裝目標

這次安裝的目標不是只讓 Hermes 指令可以跑，而是要完成一套可日常使用的 Windows 工作環境：

✅ 目標 1：CLI 可用

在 PowerShell 中可以執行 hermes、hermes setup model、hermes desktop 等指令。

✅ 目標 2：Desktop 可用

能啟動 Hermes Desktop 圖形介面，並透過模型 Provider 正常回應。

✅ 目標 3：桌面捷徑

桌面上能建立 Hermes Desktop 捷徑，必要時自動帶入 --disable-gpu 參數。

✅ 目標 4：踩雷可復原

遇到 Git stash 衝突、Provider 未設定、GPU process 錯誤時，有明確修復步驟。

本次實際路徑

Hermes 安裝位置最後確認為：C:\Users\user\AppData\Local\hermes\hermes-agent。不同電腦的使用者名稱可能不同，建議用 $env:LOCALAPPDATA 代替固定路徑。

二、本次安裝歷程摘要

開始安裝 Hermes Agent。 系統檢查 Python、Git、Node.js、ripgrep、ffmpeg 等必要依賴。
遇到 Nous Portal OAuth 登入。 出現 device code 與瀏覽器驗證，但因逾時或取消，登入失敗。
選擇 Terminal Backend。 最後採用 Local / Keep current (local)，讓 Hermes 在本機執行指令。
Gateway / Provider 尚未完成。 Desktop 可以進入，但顯示 Gateway needs setup 或 Inference not ready。
更新時遇到 Git stash 衝突。 衝突檔案包含 mixture-of-agents.md 與 slack.md。
找到真正 Git repo。 初期誤以為 AppData\Local\hermes 是 repo，後來確認真正 repo 是 AppData\Local\hermes\hermes-agent。
安裝完成。 畫面顯示 Ready to go!，CLI 安裝成功。
Desktop 啟動遇 GPU 錯誤。 顯示 GPU process isn't usable. Goodbye.，最後用 --disable-gpu 成功啟動。

三、踩雷總表：問題、原因與解法

問題畫面 / 訊息 可能原因 解決方式
Login failed: Timed out waiting for device authorization Nous Portal OAuth 授權沒有在時間內完成。 先取消登入，繼續設定。之後可改用 hermes setup model 設定 OpenRouter、OpenAI、Ollama 或重新登入 Nous Portal。
Terminal Backend 要選哪個？ Hermes 詢問指令要在哪裡執行。 一般 Windows 本機使用，選 6. Keep current (local) 或直接按 Enter。
Messaging 平台列表全部 not configured 尚未設定 LINE、Telegram、Discord、Slack 等 token。 第一次安裝先全部跳過，直接按 Enter。之後有需要再設定。
Gateway needs setup / Inference not ready Hermes 已啟動，但還沒有模型 Provider。 執行 hermes setup model，選擇 Nous Portal、OpenAI、OpenRouter、Claude 或 Ollama。
git stash apply failed after update 安裝器更新 Hermes 後，嘗試還原本機修改，但檔案衝突。 進入真正 repo：$env:LOCALAPPDATA\hermes\hermes-agent，執行 git reset --hard 與 git clean -fd。
fatal: not a git repository 進錯資料夾，AppData\Local\hermes 只是資料目錄，不是 Git repo。 改進入 AppData\Local\hermes\hermes-agent。
GPU process isn't usable. Goodbye. Electron Desktop 的 GPU 加速失敗。 用 --disable-gpu --no-sandbox 啟動，並把參數寫入桌面捷徑。

四、CLI 版本成功安裝流程

CLI 是 Hermes Agent 的基礎。Desktop 其實也依賴底層 Hermes 安裝與設定，因此建議先確定 CLI 成功，再處理 Desktop。

  1. 下載並執行官方安裝腳本。 
    Invoke-WebRequest -Uri "https://hermes-agent.nousresearch.com/install.ps1" -OutFile install.ps1
.\install.ps1
  2. 若問 Restore local changes now?

    本次因先前 stash 還原發生衝突，建議選 n，避免舊的衝突修改再次套回來。

    Restore local changes now? [Y/n]: n
  3. Terminal Backend 選 Local。

    一般個人電腦使用，選 6. Keep current (local) 或直接按 Enter。

  4. 看到 Ready to go! 即代表 CLI 安裝完成。 
    Ready to go!

hermes          Start chatting
hermes gateway  Start messaging gateway
hermes doctor   Check for issues

安裝完成後測試 CLI

hermes

若可以進入 Hermes 聊天介面，代表 CLI 已可啟動。若沒有回應，通常不是 CLI 壞掉，而是模型 Provider 尚未設定。

五、Desktop 版本啟動流程

CLI 成功後，可用下列指令啟動 Desktop：

hermes desktop

本次啟動時，畫面顯示 Desktop packaged app 已是最新，並嘗試啟動：

Launching packaged Hermes Desktop:
C:\Users\user\AppData\Local\hermes\hermes-agent\apps\desktop\release\win-unpacked\Hermes.exe

遇到 GPU 錯誤

錯誤訊息 
GPU process exited unexpectedly
GPU process isn't usable. Goodbye.

這代表 Hermes Desktop 的 Electron GUI 在 Windows 顯示卡環境中無法正常使用 GPU 加速。處理方式是改用關閉 GPU 加速的方式啟動。

關閉 GPU 加速啟動

& "$env:LOCALAPPDATA\hermes\hermes-agent\apps\desktop\release\win-unpacked\Hermes.exe" --disable-gpu --no-sandbox
本次成功解法

使用 --disable-gpu --no-sandbox 後，Hermes Desktop 已可正常運作。

六、建立桌面捷徑與 Hermes Logo

因為本次 Desktop 需要帶入啟動參數，建議不要只手動建立一般捷徑，而是用 PowerShell 自動建立，並把 Hermes 主程式 icon 設成捷徑圖示。

建立 Desktop 捷徑：含 Logo 與 GPU 參數

$desktop = [Environment]::GetFolderPath("Desktop")
$shortcutPath = Join-Path $desktop "Hermes Desktop.lnk"
$hermesExe = "$env:LOCALAPPDATA\hermes\hermes-agent\apps\desktop\release\win-unpacked\Hermes.exe"

$wsh = New-Object -ComObject WScript.Shell
$shortcut = $wsh.CreateShortcut($shortcutPath)
$shortcut.TargetPath = $hermesExe
$shortcut.Arguments = "--disable-gpu --no-sandbox"
$shortcut.WorkingDirectory = Split-Path $hermesExe
$shortcut.Description = "Hermes Desktop"
$shortcut.IconLocation = "$hermesExe,0"
$shortcut.Save()

Write-Host "已建立 Hermes Desktop 桌面捷徑"

建立 CLI 捷徑：開啟 PowerShell 並執行 hermes

$desktop = [Environment]::GetFolderPath("Desktop")
$shortcutPath = Join-Path $desktop "Hermes Agent CLI.lnk"

$wsh = New-Object -ComObject WScript.Shell
$shortcut = $wsh.CreateShortcut($shortcutPath)
$shortcut.TargetPath = "powershell.exe"
$shortcut.Arguments = "-NoExit -Command hermes"
$shortcut.WorkingDirectory = "$env:LOCALAPPDATA\hermes\hermes-agent"
$shortcut.Description = "Hermes Agent CLI"
$shortcut.Save()

Write-Host "已建立 Hermes Agent CLI 桌面捷徑"
注意

若捷徑圖示沒有立即更新，可在桌面按 F5 重新整理，或登出再登入 Windows。

七、模型 Provider / Gateway 設定

Hermes Agent 本體安裝完成後，仍需要模型 Provider 才能真正回答問題。若桌面左下角顯示 Gateway needs setup 或 Inference not ready，請設定模型。

hermes setup model
Provider是否可能付費適合情境
Nous Portal可能需要訂閱或額度官方推薦、整合度高，但需完成登入授權。
OpenAI API依 API 用量計費已經有 OpenAI API Key 的使用者。
OpenRouter依模型與用量計費想測試多種模型，彈性高。
Anthropic / Claude依 API 用量計費需要高品質文字、規劃與推理任務。
OllamaAPI 費可為 0，但吃本機效能想先免費測試，或重視本機隱私。
建議

初次使用若不想綁訂閱，可先跳過 Nous Portal，之後再用 hermes setup model 設定 OpenRouter、OpenAI 或 Ollama。

九、快速故障排除

1. 安裝更新又卡在 stash 衝突

cd "$env:LOCALAPPDATA\hermes\hermes-agent"
git status
git reset --hard
git clean -fd

重新執行安裝時，若問 Restore local changes now? [Y/n]，建議輸入 n。

2. 找不到真正 Hermes Git repo

Get-ChildItem -Path $env:LOCALAPPDATA,$env:USERPROFILE -Directory -Recurse -Force -ErrorAction SilentlyContinue | Where-Object { $_.Name -eq ".git" -and $_.FullName -match "hermes" } | ForEach-Object { Split-Path $_.FullName -Parent }

3. Desktop 閃退或 GPU 錯誤

& "$env:LOCALAPPDATA\hermes\hermes-agent\apps\desktop\release\win-unpacked\Hermes.exe" --disable-gpu --no-sandbox

4. Hermes 沒反應、等待模型太久

先不要丟太大的任務。建議先測試一句：

請用繁體中文回答一句話：你現在可以正常回應嗎？

若簡單問題也沒有回應，優先檢查 Gateway / Provider 設定。
